Detailed Description

Purpose of TB_Hamiltonian:

  • define Hamiltonian abstractly and store everything nessecary to generate matrix elements
  • avoid dependence on concrete method of diagonalization (exact or SLEPc)
  • if we want to use the same tool for many different systems (e.g. generalize to graphene or MoS2), we need to allow for different models with different numbers of neighbors, different numbers of local orbitals per atoms, etc.
  • TODO: generalize to include arbitrary potentials (as functions of positions) and magnetic field (g-factor, phase in hopping element)

-> class has to be rather abstract.

What we need:

  • function to generate row of a Hamiltonian. Or better: Block of rows corresponding to a single atom. From this we can either generate the Hamiltonian line by line or the sparse matrix form.
  • storage of atoms and neighbors list, determined by lattice
  • storage of TB parameters, determined by TB model
